Architecture

Modular Structure

Nodesmith encourages a modular approach. Each feature or domain is a self-contained module with its own routes, models, and logic.

modules/
  user/
    index.ts
    controller/
  blog/
    index.ts
    controller/

Event-Driven Communication

Modules communicate via events, not direct calls. This decouples features and enables advanced patterns like logging, monitoring, and async workflows.

eventBus.emit('user.registered', { id: 42 });
eventBus.on('user.registered', (data) => { /* ... */ });

Dependency Injection (DI)

All services and shared resources are registered in the DI container, making them easy to swap, mock, or extend.

container.register('EmailService', EmailService, { singleton: true });
const emailService = container.resolve('EmailService');

Best Practices

  • Keep modules independent and focused
  • Use events for cross-module communication
  • Register shared services in the DI container
  • Document your module's events and API
