🚀 Core Features

🎯 Auto-Discovery System

Automatically discover and register components based on folder structure. No manual registration needed!

// Drop files in folders and they're auto-registered
routes/userRoutes.ts     → HTTP routes
events/userEvents.ts     → Event handlers  
services/UserService.ts  → Business logic
middleware/auth.ts       → Custom middleware

Learn more about Auto-Discovery →

💉 Dependency Injection (DI) Container

Register and resolve services, support for singletons and transients, and more.

container.register('MyService', MyService, { singleton: true });
const myService = container.resolve('MyService');

🔗 Event Bus

Decouple modules with a global event bus. Emit and listen for events anywhere.

eventBus.emit('user.created', { id: 1 });
eventBus.on('user.created', (user) => { /* ... */ });

📦 Module Loader

Automatically loads enabled modules from configuration, validates dependencies, and supports hot-reloading in development.

const modules = loadEnabledModules();

📝 TypeScript-First

Strong typing for all core APIs, module manifests, and lifecycle hooks. Enjoy autocompletion and compile-time safety.

🛠️ CLI Tools

Powerful command-line tools for module management and discovery.

npm run cli module:list      # List all modules
npm run cli module:discover  # Show discovered components
npm run cli module:create    # Create new modules
npm run cli db:migrate       # Handle database migrations

🏗️ Structured Module System

Consistent folder structure across all modules with helpful READMEs and documentation.

  • Zero Configuration: Just drop files in folders
  • Consistent Structure: All modules follow the same pattern
  • Easy to Add: No manual registration required
  • Discoverable: CLI tools show what's available
